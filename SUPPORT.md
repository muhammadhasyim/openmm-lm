## How to Get Support for OpenMM-LM

There are two main venues for getting support: the GitHub
[discussion forum](https://github.com/muhammadhasyim/openmm-lm/discussions) and the
[issue tracker](https://github.com/muhammadhasyim/openmm-lm/issues). There is some overlap
between the two, but generally speaking the forum is for user oriented issues while the
issue tracker is for developer oriented issues. If you have a question about how to use OpenMM-LM
(including writing programs that access it through its public API), post on the forum. If
you want to suggest a change to the code, or if you think you have found a bug,
open an issue. The core developers monitor both, so don't worry if you aren't
sure which one is most appropriate for your question. We will see it either way.

You also may want to consult the documentation linked from [README.md](README.md). For the
shared OpenMM API foundation, upstream [docs.openmm.org](https://docs.openmm.org/) remains useful.

### Upstream OpenMM

Questions about unmodified upstream OpenMM behavior (without OpenMM-LM extensions) may also be
discussed on [upstream OpenMM](https://github.com/openmm/openmm/discussions), but build,
install, UMA/RPMD/`PythonForce` batching, and cavity-MD issues should stay in **this**
repository so context stays with the modified code.
